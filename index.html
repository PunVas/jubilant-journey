> DNS: Distributed Infrastructure_

Routing, Resolution, and Eventual Consistency

user@system:~$ cat /etc/resolv.conf nameserver 8.8.8.8 nameserver 1.1.1.1

> The Hierarchy (Tree of Truth)

. (ROOT) ├── .com (TLD) │ └── google.com (Authoritative)
  • Root (.): The 13 logical entry points. Knows TLDs.
  • TLD (.com): Managed by registries. Delegates downward.
  • Authoritative: The final ledger of truth.
  • Strict Delegation: Prevents single points of failure.

> The Schema: DNS Records

  • A / AAAA: The actual IP pointers (IPv4 / IPv6).
  • CNAME: Canonical Name. Aliases used for load balancing.
  • NS (Name Server): The Delegation Record. This makes the distributed hierarchy possible.
  • MX: Mail Exchange. Routes SMTP traffic.
  • TXT: Text strings. Now crucial for Security (SPF, DKIM).

> Network Layer (Port 53)

  • Default: UDP (Connectionless).
  • Why: Eliminates TCP 3-way handshake latency.
  • The Limit: 512 bytes payload limit (historically).
  • TCP Fallback: If response > 512 bytes, the TC (Truncated) bit triggers. Client retries over TCP. Also used for Zone Transfers.

> Global Scale (BGP Anycast)

  • The Myth: "There are only 13 Root Servers."
  • The Reality: 1,700+ physical servers share 13 IPs.
  • Mechanism: Border Gateway Protocol (BGP) Anycast.
  • Result: Network inherently routes queries to the geographically nearest server. Zero load balancers needed.

> Eventual Consistency & Caching

  • DNS strictly favors Availability over Consistency.
  • Caching Layers: Browser -> OS -> ISP Resolver.
  • TTL (Time To Live): Dictates cache expiration.
  • The Trade-off:
    • High TTL = Low latency, high availability.
    • Low TTL = Fast update propagation, high query load.
